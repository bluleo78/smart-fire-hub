### 1.2 각 플랫폼 상세 분석

#### 1.2.1 ImageTrend Elite

**개요**: 소방 기록 관리 시스템(RMS) 시장의 선두 주자. 사건 보고, 검사, 교육, 사전계획(Pre-Plan) 등을 통합 관리하는 클라우드 기반 플랫폼.

**핵심 기능**:
- **사건 보고(Incident Reporting)**: 160개 이상의 사전 제작 보고서와 Ad-hoc 리포팅 기능. 고급 쿼리 기능 포함
- **NERIS/NFIRS 준수**: 내장 규칙, 자동화, 교육을 통한 보고 표준 준수. 2026년 NERIS 전환 완벽 지원
- **오프라인 기능**: Elite Field Inspections로 오프라인에서 건물 검사/사전계획 수행 가능. 인터넷 연결 시 자동 동기화
- **시스템 통합**: CAD, 스케줄링, 빌링 시스템과 원활한 연동
- **데이터 관리**: 클라우드 기반 위치/점유자/검사 데이터 관리

**취급 데이터 유형**:
- 화재 사건 보고서, EMS/구급 기록(ePCR)
- 건물 검사/점유 데이터, 사전계획(Pre-Plan)
- 인력 관리(교육, 자격증, 근무이력)
- 소화전(Hydrant) 데이터
- 장비/차량 데이터

**강점**: NERIS 전환 지원 최전선, 오프라인 현장 기능, 방대한 기존 보고서 라이브러리
**약점**: 자체 CAD 미보유(연동 의존), AI/ML 기능 제한적

---

#### 1.2.2 ESO (구 ESO Solutions)

**개요**: 소방, EMS, 병원을 연결하는 데이터 에코시스템 구축을 목표로 하는 플랫폼. 사건 보고부터 인력 관리까지 올인원 솔루션 제공.

**핵심 기능**:
- **Fire Incidents**: 웹 기반 사건 보고. 사용자 수 무제한. Progressive Validation 시퀀스로 NFIRS/NERIS 보고서 작성 가이드
- **Personnel Management**: 교육, 자격증, 경력, 인구통계, 면역, 부상/노출 관리. 자격증 만료 알림 기능
- **ESO Hydrants**: 관할 구역 내 소화전 정보 관리
- **ESO Permits**: 허가 정보 수집 및 인쇄/이메일 발송
- **ESO Scheduling**: 유연한 직원 스케줄링, 유닛/스테이션/교대별 인력 추적

**취급 데이터 유형**:
- 화재 사건 기록, EMS 환자 기록(ePCR)
- 인력/교육/자격증 데이터
- 소화전 데이터, 허가 데이터
- 스케줄링/근무 데이터
- +1 코드 및 NFIRS 코드 지원

**강점**: EMS-소방-병원 연속성(Continuum of Care) 데이터 연결, Progressive Validation으로 데이터 정확성 향상
**약점**: GIS 기능 제한적, 실시간 분석 기능 부족

#### 1.2.3 FirstWatch

**개요**: 실시간 상황 인식(Situational Awareness), 데이터 감시(Surveillance), 조기 경보(Early Warning) 전문 소프트웨어. 911, EMS, 소방, 법집행, 공중보건 팀을 위한 대시보드 솔루션.

**핵심 기능**:
- **실시간 대시보드**: 상황 인식, 응답 시간, KPI, 임상/운영 성과 모니터링
- **자동 경보**: 생물테러, 팬데믹/공중보건 감시를 위한 자동화된 알림 시스템
- **신드로믹 서베일런스**: 잠재적 전염병 발생부터 생물테러까지 조기 이벤트 감지
- **지역 데이터 집계**: 다수 기관의 데이터를 집계하여 광역 분석 지원
- **데이터 소스 연동**: CAD, ProQA, ePCR, RMS, 병원 데이터를 실시간 모니터링
- **수요 분석(Demand Analysis)**: 수요 차트 생성 및 인력 배치 최적화

**취급 데이터 유형**:
- CAD 출동 데이터 (실시간)
- ePCR 환자 데이터 (실시간)
- RMS 기록 데이터
- 병원 데이터
- 공중보건 감시 데이터

**강점**: 실시간 분석 특화, 다기관 데이터 집계, 공중보건 감시 통합
**약점**: RMS 자체 미보유, 사건 보고 기능 없음(분석 전문)

---

#### 1.2.4 Deccan International (ADAM/LiveMUM)

**개요**: 소방/EMS 리더십을 위한 의사결정 지원 소프트웨어 전문 기업. 배치 최적화와 응답 시간 분석에 특화.

**핵심 제품군**:
- **ADAM (Apparatus Deployment Analysis Module)**: "What-if" 예측 모델링 도구. 과거 CAD 데이터, GIS 지도 데이터, 예측 알고리즘을 사용하여 배치 변경이 응답 시간과 가용성에 미치는 영향을 분석
- **LiveMUM (Live Move-Up Module)**: 실시간 커버리지 모니터링, 알림, 이동 배치(Move-Up) 추천. 디스패처에게 자동화된 최적 이동 배치 추천 제공
- **BARB**: 분석 및 보고 도구
- **DiVa**: 데이터 시각화 도구

**취급 데이터 유형**:
- 과거 CAD 데이터 (출동 이력)
- GIS 지도 데이터 (도로 네트워크, 응답 구역)
- 차량/장비 배치 데이터
- 실시간 유닛 위치/상태 데이터

**강점**: 배치 최적화 분야 최고 전문성, 강력한 GIS 분석, 예산/자원 정당화 도구
**약점**: 기록 관리(RMS) 기능 없음, 사건 보고 기능 없음

---

#### 1.2.5 Tyler Technologies (구 New World Systems)

**개요**: 1981년 설립, 2015년 Tyler Technologies에 인수. 공공안전 분야 CAD/RMS 통합 솔루션의 대표 기업.

**핵심 기능**:
- **New World CAD**: 콜 테이커와 디스패처를 위한 통합 CAD 시스템. Microsoft Windows 환경, 커맨드라인 입력, 드래그앤드롭 디스패치, 통합 지도
- **Enterprise RMS**: 법집행, 소방, EMS를 위한 통합 기록 관리 시스템
- **Fire RMS**: 소방 전용 기록 관리. 소화전 위치, 사전계획 접근 지원
- **모바일 컴퓨팅**: 현장 요원을 위한 모바일 애플리케이션
- **Aegis 플랫폼**: 법집행, 소방/EMS RMS, CAD, 모바일 컴퓨팅 통합

**취급 데이터 유형**:
- CAD 출동 데이터, 사건 기록
- 소화전 위치 데이터
- 사전계획(Pre-Plan) 데이터
- 지도/GIS 데이터
- 인력/장비 데이터

**강점**: CAD-RMS 완전 통합, 대규모 설치 기반, 안정적 플랫폼
**약점**: 레거시 아키텍처(온프레미스 기반 다수), 클라우드 전환 진행 중

#### 1.2.6 Hexagon Safety

**개요**: 전 세계 10억 명 이상의 인구를 보호하는 공공안전 솔루션 글로벌 리더. AI 기반 데이터 수집, 관리, 분석, 협업, 대응 플랫폼 제공.

**핵심 기능**:
- **HxGN OnCall Dispatch (CAD)**: 대응 계획 생성, 유닛 추천, 사건 경계 설정, 병원 라우팅. 스테이션 알림, 인쇄, 페이징 시스템 연동
- **공공안전 분석**: 디스패치, 기록, 사건 데이터를 통합하여 단일 정보 소스(Single Source of Truth) 제공. 맞춤형 보고서 및 대시보드 생성
- **GIS 통합 매핑**: 대응자에게 건물/이벤트 현장의 상세 청사진 제공하는 인터랙티브 맵
- **자원 최적화**: 실시간 데이터를 활용한 자원 최적화, 위험 식별, 사전 계획 수립
- **NERIS 준수**: First Due와 협력하여 NERIS 보고 지원

**취급 데이터 유형**:
- CAD 출동/디스패치 데이터
- GIS/지리공간 데이터
- 사건 기록 데이터
- 실시간 유닛 위치/상태
- 건물/시설 데이터
- 비디오/센서 데이터

**강점**: GIS 기술력 세계 최고, CAD 시장 점유율 높음, AI 기반 분석
**약점**: 고가의 라이선스 비용, 복잡한 구현/운영

#### 1.2.7 Mark43

**개요**: AWS GovCloud에서 처음부터 구축된 유일한 클라우드 네이티브 공공안전 플랫폼. 경찰-소방 통합 운영 지원.

**핵심 기능**:
- **클라우드 네이티브 RMS**: 체포/범죄 보고서 작성 시간 50-80% 감소. NIBRS 준수 기반 설계
- **CAD**: 경찰-소방 통합 디스패치. 정보 공유 장벽 제거, 빠르고 조율된 대응
- **OnScene (모바일)**: 실시간 사건 인텔리전스를 현장 대응자에게 전달. 순찰차, 소방차, 도보, 대규모 비상 대응 모두 지원
- **Insights (분석)**: 고급 분석 및 대시보드
- **스마트 자동화**: 자원 배정 및 수사 진행 자동화

**취급 데이터 유형**:
- 사건/범죄 기록 데이터
- CAD 출동 데이터
- 증거/재산 관리 데이터
- 인력/자원 데이터
- 실시간 현장 데이터

**강점**: 완전한 클라우드 네이티브 아키텍처, 경찰-소방 통합, 현대적 UX
**약점**: 소방 전용 기능 상대적으로 제한(법집행 중심 시작), NFIRS/NERIS 보고 기능 발전 중

#### 1.2.8 First Due

**개요**: 소방/EMS 운영 전체를 하나의 플랫폼으로 통합. 사전계획, NFIRS, ePCR, 예방, 스케줄링, 자산관리, 소화전, 교육, 지역사회 참여, 모바일 대응 등 포함.

**핵심 기능**:
- **사전계획(Pre-Incident Planning)**: 주소 입력만으로 사전계획 생성 시작. 건물 치수, 화재 부하, 필요 유량 등 구조 상세 정보 추가. 유틸리티 차단, 소방 연결, 특수 위험물, 위험 물질을 지도에 직접 표시
- **화재 예방/검사**: 예방 중심 워크플로에 완전 통합
- **모바일 앱**: 현장에서 실시간 사전계획 데이터 접근
- **NERIS 보고**: Hexagon과 협력하여 NERIS 보고 지원
- **응급 대응 가이드**: 화학물질 사고 시 원클릭으로 응급 대응 가이드 접근

**취급 데이터 유형**:
- 사전계획/건물 데이터
- 검사/예방 데이터
- NFIRS/NERIS 사건 보고
- ePCR 데이터
- 인력/스케줄링 데이터
- 소화전/자산 데이터
- 교육 기록

**강점**: 소방 전용 올인원 플랫폼, 우수한 사전계획 기능, 현장 중심 설계
**약점**: 대규모 기관에서의 확장성 검증 필요, CAD 자체 미보유

#### 1.2.9 Motorola Solutions (CommandCentral)

**개요**: 공공안전 최초의 클라우드 네이티브 엔드투엔드 지휘 센터 소프트웨어 슈트. 911 콜 처리부터 사건 종결까지 데이터 흐름 통합.

**핵심 기능**:
- **CommandCentral Aware**: 실시간 클라우드 기반 상황 인식. GIS 레이어 위에 지도 기반 인텔리전스 + 스트리밍 비디오 결합
- **통합 데이터 플랫폼**: 911 콜 처리, 디스패치, 비디오 분석, 현장 보고서, 기록, 증거, 지역사회 참여 데이터를 수집하여 360도 사건 뷰 제공
- **비디오/분석**: 사건 인근 비디오 피드 실시간 접근. 비디오 및 차량번호 인식(LPR) 분석
- **소방 전용**: 제어실 효율성, 실시간 상황 인식을 통한 소방관 지원

**강점**: 음성/비디오/데이터 통합 에코시스템, 강력한 하드웨어-소프트웨어 통합
**약점**: 벤더 락인(Vendor Lock-in) 위험, 고가의 통합 솔루션

---

#### 1.2.10 RapidSOS

**개요**: NG911 응급 대응 데이터 플랫폼. PSAP에 실시간 위치 정보 및 보충 응급 데이터 제공. Apple, Google, MedicAlert, Uber 등과 파트너십.

**핵심 기능**:
- **위치 정보**: 스마트폰 기반 하이브리드 위치 정보 제공 (앱 불필요)
- **NENA i3 준수**: Location Information Server (LIS) 및 Additional Data Repository (ADR)
- **RapidSOS Portal**: 브라우저 기반 보충 응급 데이터 접근 (무료)
- **API 통합**: 기존 공공안전 소프트웨어와 연동

**강점**: 무료 NG911 데이터 접근, 3,000+ 기관 사용, 광범위한 데이터 파트너십
**약점**: 분석/보고 기능 없음, CAD/RMS 연동 의존

#### 1.2.11 NFORS (National Fire Operations Reporting System)

**개요**: 국제 공공안전 데이터 연구소(I-PSDI)가 운영하는 소방 운영 데이터 수집 및 분석 시스템. StatEngine 기반.

**핵심 기능**:
- **자동 데이터 수집**: CAD/RMS에서 직접 데이터 수집. 추가 입력 불필요
- **실시간 분석**: 데이터 수집 후 즉시 분석. 약 5분마다 업데이트
- **맞춤형 대시보드**: 부서별 맞춤 대시보드. 필터링 및 드릴다운 분석
- **노출 추적**: 소방관 개인 노출 이력 데이터베이스 (모바일 앱)
- **운영 효과성 평가**: 가용성, 역량, 운영 효과성이 결과에 미치는 영향 평가

**강점**: 무입력 자동 데이터 수집, 실시간 운영 분석, 노출 추적 기능
**약점**: 대중적 인지도 낮음, 제한된 기관 사용

## 2. 소방 데이터 표준 및 프로토콜

### 2.1 NFIRS (National Fire Incident Reporting System)

> **중요**: NFIRS는 2026년 2월부로 종료(Sunset)되었으며, NERIS로 완전 전환됨

**개요**: 미국 소방관리국(USFA)이 운영하던 국가 화재사건 보고 시스템. 1976년 시작, Version 5.0이 마지막 버전.

**모듈 구조**:

| 모듈 | 코드 | 설명 | 필수 여부 |
|------|------|------|-----------|
| **기본 모듈 (Basic)** | NFIRS-1 | 모든 사건의 일반 정보 (누가/어디서/언제) | 필수 |
| **화재 모듈 (Fire)** | NFIRS-2 | 화재 사건 상세 기술 | 화재 시 필수 |
| **구조물 화재 모듈 (Structure Fire)** | NFIRS-3 | 구조물 화재 상세 (화재 모듈과 함께 사용) | 구조물 화재 시 |
| **민간인 화재 사상자 모듈** | NFIRS-4 | 민간인/타 응급요원 부상/사망 보고 | 사상자 발생 시 |
| **소방관 사상자 모듈** | NFIRS-5 | 소방관 부상/사망 보고 | 소방관 사상 시 |
| **EMS 모듈** | NFIRS-6 | 의료 사건 보고 (1차 환자 치료 담당 시) | EMS 제공 시 |
| **위험물 모듈 (HazMat)** | NFIRS-7 | 55갤런 이상 유출/방출 또는 특수 HazMat 조치 | 해당 시 |
| **산불 모듈 (Wildland Fire)** | NFIRS-8 | 산불/식생 화재 보고 | 산불 시 |
| **장비/자원 모듈 (Apparatus)** | NFIRS-9 | 출동 장비/자원 보고 | 선택 |
| **인력 모듈 (Personnel)** | NFIRS-10 | 사건 관련 인력 추적 | 선택 |
| **방화 모듈 (Arson)** | NFIRS-11 | 의도적 방화 추가 정보 | 방화 시 |

**설계 특징**:
- 모듈식 설계로 사건 유형에 따라 필요한 데이터만 수집
- 기본 모듈은 모든 출동에 필수, 추가 모듈은 사건 유형별 조건부 적용
- 표준화된 코드 체계(사건 유형, 발화 원인, 재산 유형 등)

### 2.2 NERIS (National Emergency Response Information System)

> **현재 상태**: 2026년 2월 1일부로 미국의 유일한 화재/전재해 데이터 시스템으로 운영 중

**개요**: NFIRS를 대체하는 차세대 국가 응급 대응 정보 시스템. USFA, DHS S&T, FSRI(UL Research Institutes)가 공동 개발. 2024년 가을 v1.0 출시.

**핵심 데이터 스키마**:

| 스키마 | 설명 | 핵심 요소 |
|--------|------|-----------|
| **Entity** | 소방서 엔티티 정의 | 부서 자원, 역량, 조직 구조, 장비, 인력 |
| **Dispatch (CAD)** | CAD/RMS에서 수집되는 디스패치 데이터 | 출동 시간, 유닛, 위치, 상태 변경 |
| **Incident** | 사건별 수집 정보 | 화재, 위험 상황, 의료, 신규 위험, 위험 감소, 구조 등 |

**사건 유형 체계** (NFIRS 대비 혁신적 변경):
- **Primary Group**: 대분류 (화재/의료/구조 등)
- **Sub-Group**: 중분류 (실내화재/실외화재, 외상/질병 등)
- **Incident Type**: 세분류 (구조물 화재, 교통사고 등)
- 하나의 사건에 복수 유형 지정 가능 (NFIRS와 다른 핵심 차이)

**NFIRS 대비 주요 변경사항**:
- 더 상세하고 결과 중심적(Outcome-focused) 데이터 수집
- 화재 역학(Fire Dynamics) 관련 포괄적 데이터 요구
- NEMSIS, CAD 등 타 데이터 표준과의 호환성
- 모듈식 아키텍처와 API 통합으로 빠른 업데이트 가능
- 7개 핵심 사건 범주 중심의 유연한 분류 체계

**기술 인프라**:
- GitHub 오픈소스 프레임워크: `github.com/ulfsri/neris-framework`
- 모듈(데이터 요소 정의)과 값 세트(선택지) 분리
- API 기반 데이터 교환
- 클라우드 기반 플랫폼

---

### 2.3 NEMSIS (National EMS Information System)

**개요**: 미국 전역의 병원 전 환자 기록(ePCR) 수집, 공유, 평가를 위한 범용 표준. 현재 v3.5.x 운영 중.

**데이터 구조**:

| 데이터 세트 | 약칭 | 설명 |
|-------------|------|------|
| **EMS (ePCR)** | eRecord | 개별 환자 접촉 기록 |
| **Demographic (Agency)** | dAgency | 기관 인구통계 정보 |
| **StateDataSet** | SDS | 주(州) 수준 데이터 세트 |

**명명 규칙**:
- 섹션 설명어 + 마침표 + 데이터 요소 번호 (예: `dAgency.01`)
- 코드: 7자리 숫자, 2-2-3 패턴 (섹션-요소-값)

**값 처리 체계**:
- `Not Applicable`: 해당 없음
- `Not Recorded`: 해당하나 미기록
- `Not Reporting`: 기관/주에서 미수집

**기술 구현**:
- XSD 기반 스키마 정의
- Microsoft SQL Server 데이터베이스 모델
- XML 데이터 교환

### 2.4 CAD 데이터 표준

**CAD 개요**: PSAP(공공안전 응답 포인트)의 핵심 애플리케이션. 법집행, 소방, EMS 사건을 최초 신고부터 종결까지 관리.

**관련 표준 기관**:
- **APCO International**: ANSI 인증 표준 개발 기관. 통합 CAD 기능 요구사항(UCADFR) 프로젝트 추진
- **NENA**: 100개 이상의 표준 발행. EIDO(Emergency Incident Data Object) 표준 발행(2022년 10월) -- 응급 사건 데이터 교환의 사실상 표준
- **NHTSA**: CAD 상호운용성 전략 연구

**CAD 핵심 데이터 요소**:

| 데이터 범주 | 주요 항목 |
|-------------|-----------|
| **사건 정보** | 사건 번호, 유형, 우선순위, 생성 시간 |
| **위치 정보** | 주소, 좌표, 교차로, 관할 구역 |
| **시간 스탬프** | 콜 수신, 디스패치, 출동, 현장 도착, 현장 출발, 완료 |
| **유닛 정보** | 유닛 ID, 유형, 스테이션, 상태 변경 이력 |
| **디스패치 정보** | 디스패처 ID, 추천 유닛, 알람 레벨 |
| **현장 정보** | 현장 상황, 조치 내용, 결과 |

**상호운용성 과제**:
- 수십 년간 공통 표준 미합의 상태
- APCO/NENA가 표준 설정하나 강제력 없음
- CAD-to-CAD 상호운용성은 기술 문제가 아닌 합의 문제

### 2.5 ISO/PPC (Public Protection Classification)

**개요**: Verisk(구 ISO)의 FSRS(Fire Suppression Rating Schedule)를 기반으로 지역사회의 화재 진압 역량을 평가하여 1~10 등급(PPC) 부여. 보험 요율 산정에 직접 영향.

**평가 영역 및 배점**:

| 평가 영역 | 배점 | 핵심 평가 항목 |
|-----------|------|----------------|
| **응급 통신 시스템** | 10점 | 911 시스템, 디스패치 시설, 통신 장비, 인력 |
| **소방서** | 50점 | 인력, 역량, 교육, 장비, 배치, 지리적 분포 |
| **급수 시스템** | 40점 | 급수 공급량, 소화전 분포, 소화전 검사/유지보수 |
| **지역사회 위험 감소** | +5.5점(가산) | 화재 예방, 공공 안전 교육, 화재 조사 |

**참조 표준**: NFPA, AWWA(미국상수도협회), APCO International의 국가 수용 표준 활용

**데이터 요구사항**:
- 소방서: 인력 배치 계획, 장비 목록, 교육 기록, 응답 시간 데이터
- 급수: 소화전 위치/유량/검사 기록, 급수 시스템 용량
- 통신: 디스패치 시간 데이터, 장비 사양, 인력 배치
- 위험 감소: 검사 기록, 교육 프로그램, 화재 조사 기록

### 2.7 CFAI/CPSE 인증 데이터 요구사항

**개요**: 공공안전 우수성 센터(CPSE)의 화재 및 응급 서비스 인증(CFAI Accreditation). "지역사회 지향적, 데이터 중심적, 결과 중심적" 기관 인증.

**데이터 보고 요구사항**:
- 모든 시간 데이터: **90th 백분위수** 기준 보고
- 초기 인증: **3년** 성과 데이터 추적
- 재인증: **5년** 성과 데이터 추적
- 인구 밀도별 구분 보고: 도시(2,500명 이상)/농촌(2,500명 미만)

**성과 데이터 차트 요소**:
- 콜 처리 시간, 출동 시간(Turnout Time), 이동 시간(Travel Time)
- 총 응답 시간, 유닛별/구역별/교대별 분석
- 기준선 성과 식별 및 목표 달성 여부 추적

## 3. 핵심 소방 데이터 도메인

### 3.1 사건 대응 데이터 (Incident Response)

**가장 핵심적인 데이터 도메인**. 모든 소방 분석의 기반.

| 데이터 항목 | 설명 | 핵심 KPI |
|-------------|------|----------|
| **콜 처리 시간** | 콜 수신 ~ 1차 유닛 디스패치 | NFPA 기준: 64초 이내 (95%) |
| **출동 시간 (Turnout Time)** | 디스패치 ~ 유닛 출동 시작 | NFPA 1710: EMS 60초 / 화재 80초 |
| **이동 시간 (Travel Time)** | 출동 시작 ~ 현장 도착 | 도시 지역: 4분 이내 (90%) |
| **총 응답 시간 (Response Time)** | 콜 수신 ~ 현장 도착 | 도시 화재: 약 5분 |
| **최초 전체 알람 집결 시간** | 디스패치 ~ 전체 인력 집결 | NFPA 1710: 480초/8분 (90%) |
| **사건 유형** | 화재, EMS, 구조, HazMat, 기타 | 유형별 분포 분석 |
| **유닛 출동 정보** | 출동 유닛, 인원, 장비 | 유닛당 출동 횟수, 가용성 |
| **사건 결과** | 인명피해, 재산피해, 진압 결과 | 피해 최소화율, 구조 성공률 |

**응답 시간 벤치마크 (NFPA 1710 기준)**:

```
콜 수신 ──→ 디스패치 ──→ 출동 시작 ──→ 현장 도착 ──→ 전체 집결
   │            │             │             │             │
   ├── 64초 ──→│             │             │             │
   │   (콜처리)  ├── 80초 ──→│             │             │
   │            │  (출동시간)  ├──── 240초 ──→│             │
   │            │             │  (이동시간)   │             │
   │            ├─────────── 320초 ──────→│             │
   │            │      (1차 엔진 도착)       │             │
   │            ├──────────────── 480초 ────────────→│
   │            │          (전체 알람 집결)                │
```

## 4. 소방 데이터 허브의 핵심 차별화 요소

### 4.1 시장 공백 분석 (Gap Analysis)

현재 시장의 핵심 문제점:

| 문제 | 현재 상황 | 기회 |
|------|-----------|------|
| **데이터 사일로** | 각 시스템(CAD, RMS, GIS, 인사, 장비)이 분리 운영 | 통합 데이터 허브로 단일 정보 소스 제공 |
| **수동 데이터 입력** | 소방관이 직접 데이터 입력 → 부담, 오류 | ETL 자동화로 다중 소스 자동 수집/정합 |
| **제한적 분석** | 대부분 기술적 보고(Descriptive) 수준 | 예측적/처방적 분석(Predictive/Prescriptive) 제공 |
| **표준 전환 혼란** | NFIRS→NERIS 전환으로 기관 혼란 | NERIS 네이티브 + 레거시 데이터 마이그레이션 지원 |

| **다국어 지원** | 한국어 UI/보고서 + 영어 국제 표준 호환 |

### 4.9 기술 아키텍처 권장사항

**Smart Fire Hub의 현재 기술 스택과의 정합성**:

| 현재 스택 | 소방 도메인 적용 |
|-----------|-----------------|
| **Spring Boot + jOOQ** | NERIS 스키마 기반 도메인 모델, 다중 테넌트 데이터 관리 |
| **React + TanStack Query** | 실시간 대시보드, GIS 맵 통합(Leaflet/MapLibre), KPI 시각화 |
| **Claude Agent SDK + MCP** | 소방 도메인 특화 AI 에이전트 (36종 MCP 도구를 소방 분석 도구로 확장) |
| **PostgreSQL** | `public` 스키마(메타데이터) + `data` 스키마(소방 데이터) 구조 활용 |
| **Flyway** | NERIS 스키마 버전 관리, 데이터 마이그레이션 |
| **ETL 파이프라인** | CAD/RMS/GIS 데이터 소스별 커넥터, 스케줄 기반 및 실시간 수집 |

**추가 권장 기술**:

| 기술 영역 | 권장 사항 |
|-----------|-----------|
| **GIS 엔진** | PostGIS(PostgreSQL), MapLibre GL JS(프론트엔드) |
| **실시간 처리** | Server-Sent Events(현재 AI Agent에서 사용 중) 확장, WebSocket |
| **데이터 레이크** | 대용량 과거 데이터를 위한 Parquet/Delta Lake 고려 |
| **ML 파이프라인** | Python 기반 ML 모델 서빙 (FastAPI + scikit-learn/XGBoost) |
| **시계열 DB** | 센서/IoT 데이터를 위한 TimescaleDB(PostgreSQL 확장) 고려 |

> **보고서 요약**: 소방 데이터 플랫폼 시장은 RMS(ImageTrend, ESO), CAD(Tyler, Hexagon), 실시간 분석(FirstWatch), 배치 최적화(Deccan) 등 개별 영역에 특화된 솔루션이 주를 이루며, 이들을 통합하는 데이터 허브 플랫폼은 부재한 상황이다. NFIRS에서 NERIS로의 전환(2026년 2월 완료)은 새로운 데이터 표준에 최적화된 플랫폼에 대한 수요를 창출하고 있다. 한국 시장은 국가화재정보시스템, 소방안전 빅데이터 플랫폼 등 개별 시스템이 존재하나 통합 데이터 허브는 부재하다. Smart Fire Hub는 ETL 파이프라인을 통한 다중 소스 통합, AI 에이전트 기반 분석, NERIS/한국 표준 네이티브 지원을 차별화 축으로 이 시장 공백을 공략할 수 있다.
