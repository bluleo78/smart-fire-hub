# AI Native Mode 설계 스펙

> **작성일**: 2026-03-28
> **Phase**: 6 (AI Chat Generative UI)
> **로드맵 항목**: 별도 추가 예정 (6-1 이후)

---

## 1. 개요

AI Native 모드는 기존 AI Chat 모드(side, floating, fullscreen)에 추가되는 네 번째 모드로, **AI가 관리하는 페이지 기반 위젯 캔버스**를 중심으로 한 경험을 제공한다.

- 캔버스가 전체 영역을 차지하고, 채팅은 하단 오버레이로 동작
- 캔버스는 **페이지 단위**로 관리되며, AI가 페이지 생성/전환/위젯 배치를 자율 결정
- 사용자는 페이지 이동, 위젯 닫기/크기 조절로 개입 가능
- 세션 전환은 채팅 헤더 드롭다운으로 처리 (세션 사이드바 없음)

---

## 2. 모드 순환

기존 모드 순환에 native를 추가한다:

```
side → floating → fullscreen → native → closed
```

- AIStatusChip 클릭으로 순환
- Cmd/Ctrl+K 토글은 기존과 동일 (open/close)
- localStorage에 모드 저장 (기존 `ai-mode` 키 재활용)

## 3. 레이아웃 구조

### 3.1 기본 상태 (채팅 최소화)

```
┌─────────────────────────────────────────────────┐
│  사이드바 │                                      │
│          │  ┌─────────────┬─────────────┐       │
│          │  │  위젯 A     │  위젯 B     │       │
│          │  │  (차트)     │  (테이블)   │       │
│          │  │             │             │       │
│          │  ├─────────────┴─────────────┤       │
│          │  │  위젯 C                   │       │
│          │  │                           │       │
│          │  └───────────────────────────┘       │
│          │                                      │
│          │  ← ● ○ ○ → (1/3)                    │
│          │  ┌───────────────────────────┐       │
│          │  │ [ 메시지 입력 ...   ] [전송]│       │
│          │  └───────────────────────────┘       │
└─────────────────────────────────────────────────┘
```

- 캔버스: 메인 영역 100%
- 페이지 인디케이터: 캔버스 하단 (도트 + 좌우 화살표 + 페이지 번호)
- 채팅 입력 바: 최하단 고정 (1줄)
- 세션 사이드바: 없음
- 네비게이션 사이드바: 기존 그대로 유지

### 3.2 채팅 포커스 상태 (오버레이)

```
┌─────────────────────────────────────────────────┐
│  사이드바 │  (캔버스 - 반투명 딤 처리)            │
│          │  ┌─────────────┬─────────────┐       │
│          │  │  위젯 A     │  위젯 B     │       │
│          │  │             │             │       │
│          │  ┢━━━━━━━━━━━━━━━━━━━━━━━━━━━┪       │
│          │  ┃  채팅 오버레이             ┃       │
│          │  ┃  [Session #3 ▾]           ┃       │
│          │  ┃  ──────────────────────── ┃       │
│          │  ┃  AI: 차트를 만들었습니다    ┃       │
│          │  ┃  User: 테이블도 보여줘     ┃       │
│          │  ┃  ──────────────────────── ┃       │
│          │  ┃  [ 메시지 입력 ...] [전송]  ┃       │
│          │  ┗━━━━━━━━━━━━━━━━━━━━━━━━━━━┛       │
└─────────────────────────────────────────────────┘
```

- 채팅 입력 클릭 → 채팅 패널이 하단에서 50~60% 높이로 슬라이드 업
- 캔버스는 반투명 딤 처리 (위젯 유지, 상호작용 차단)
- 닫기: 메시지 전송 후 또는 딤 영역 클릭 시 최소화
- 세션 전환: 채팅 헤더의 세션 드롭다운 (검색 + 새 세션 생성)

## 4. 캔버스 페이지 시스템

### 4.1 페이지 개념

캔버스는 **페이지** 단위로 위젯을 그룹화한다. 각 페이지는 독립적인 위젯 레이아웃을 가진다.

- AI가 대화 흐름에 따라 새 페이지를 생성하거나 기존 페이지의 위젯을 수정
- 사용자는 도트 인디케이터 또는 좌우 화살표로 페이지 간 이동
- AI가 페이지 전환 시 상단 토스트로 알림 ("AI가 페이지 2로 전환했습니다")

### 4.2 페이지 생명주기

```
새 분석 요청 → AI가 새 페이지 생성 + 위젯 배치 + 자동 전환
기존 분석 보완 → AI가 현재 페이지에 위젯 추가/교체
페이지 내 모든 위젯 닫기 → 빈 페이지 자동 삭제
```

### 4.3 AI가 부여하는 페이지 라벨

AI가 페이지 생성 시 이름을 부여한다 (예: "출동 현황", "응답 시간 분석"). 도트 인디케이터 hover 시 라벨 표시.

## 5. 위젯 그리드 시스템

### 5.1 AI 주도 유연 레이아웃

AI가 위젯마다 크기를 자유롭게 지정한다. 고정 그리드가 아닌, AI 판단 기반의 유연 배치.

**레이아웃 옵션:**

```typescript
interface CanvasLayout {
  width: "full" | "half" | "third";
  height: "full" | "half" | "third";
  replace?: string;  // 기존 위젯 ID — 해당 슬롯 교체
}
```

**배치 예시:**
- 테이블 1개 → `full × full`
- 차트 + 테이블 비교 → 각각 `half × full`
- 요약 카드 3개 → 각각 `third × half`

### 5.2 위젯 배치 애니메이션

위젯이 캔버스에 배치될 때 **슬라이드인** 효과를 적용한다:

- **새 위젯 추가**: 하단에서 위로 슬라이드하며 등장 (duration: 300ms, ease-out)
- **위젯 교체**: 기존 위젯이 위로 슬라이드 아웃 → 새 위젯이 아래에서 슬라이드 인
- **페이지 전환**: 좌우 슬라이드 (이전 페이지 왼쪽으로 퇴장, 새 페이지 오른쪽에서 진입)
- **위젯 제거(X 클릭)**: 아래로 슬라이드 아웃 + 페이드 아웃
- **재배치(크기 변경)**: 기존 위젯들이 부드럽게 이동하며 자리 재조정 (CSS transition)

### 5.3 사용자 조작

- 위젯 우상단 `X` 버튼으로 닫기
- 위젯 경계 드래그로 크기 미세 조절

### 5.3 AI 제어

- 새 위젯 추가: 현재 페이지 또는 새 페이지에 배치
- 기존 위젯 교체: `replace` 필드로 특정 위젯 슬롯 교체
- 새 페이지 생성: 새로운 분석 주제일 때 페이지를 추가하고 자동 전환

## 6. AI 에이전트 변경

### 6.1 도구 스키마 확장

기존 위젯 도구(`show_chart`, `show_table`, `show_dataset`, `navigate_to`)에 선택적 `canvas` 필드를 추가한다:

```typescript
show_chart({
  chartType: "bar",
  data: [...],
  canvas: {                    // 선택적
    width: "half",
    height: "full",
    page: "new",               // "new" = 새 페이지, "current" = 현재 페이지 (기본)
    pageLabel: "출동 현황",     // 새 페이지일 때 라벨
    replace: "widget-id-1"     // 선택적 — 기존 위젯 교체
  }
})
```

### 6.2 렌더링 규칙

- `canvas` 필드 유무와 관계없이:
  - **side / floating / fullscreen 모드** → 항상 인라인 렌더링
  - **native 모드** → 캔버스 페이지에 배치 (canvas 필드의 레이아웃 힌트 사용)
- AI는 모드를 인식하지 않아도 됨. 프론트엔드가 모드에 따라 렌더링 위치를 결정

### 6.3 시스템 프롬프트

변경 없음. AI가 도구 호출 시 `canvas` 필드 포함 여부를 자율 판단한다.

## 7. 캔버스 레이아웃 결정 흐름

### 7.1 상태 전이

```
native 모드 진입
  ├── 캔버스 페이지 캐시 있음? → 마지막 활성 페이지 그대로 표시
  └── 캐시 없음 + 인라인 위젯 있음?
      ├── 있음 → AI(haiku)에게 레이아웃 요청 → 페이지 생성 + 배치 → 캐시 저장
      └── 없음 → 빈 캔버스 플레이스홀더

native 모드 중 새 위젯 발생
  ├── canvas.page = "current" → 현재 페이지에 추가, AI(haiku)로 재배치
  └── canvas.page = "new" → 새 페이지 생성 + 위젯 배치 + 자동 전환
```

### 7.2 레이아웃 AI 호출 (경량)

- **입력**: 현재 위젯 목록 (타입, 데이터 요약) + 캔버스 크기
- **출력**: 각 위젯의 `{ width, height }` 배열
- **모델**: haiku (빠른 응답 우선)
- **폴백**: AI 호출 실패 시 규칙 기반 자동 배치 (1개=full, 2개=half+half, ...)

### 7.3 캐시

- 세션 단위로 캔버스 페이지 목록 + 각 페이지의 위젯 배치를 메모리(React state)에 보관
- 모드 전환(native ↔ 다른 모드) 시에도 세션 내에서 유지
- 새 세션 시작 시 캐시 초기화

## 8. 프론트엔드 컴포넌트 구조

### 8.1 새 컴포넌트

| 컴포넌트 | 역할 |
|----------|------|
| `AINativeMode.tsx` | native 모드 컨테이너 (캔버스 + 채팅 오버레이 오케스트레이션) |
| `CanvasArea.tsx` | 페이지 기반 위젯 캔버스 (현재 페이지의 위젯 그리드 렌더링) |
| `CanvasPage.tsx` | 개별 페이지 컨테이너 (위젯 배치 + 전환 애니메이션) |
| `CanvasWidget.tsx` | 캔버스 위젯 래퍼 (크기 조절, 닫기 버튼) |
| `PageIndicator.tsx` | 하단 도트 인디케이터 (● ○ ○ + 좌우 화살표 + 페이지 번호) |
| `ChatOverlay.tsx` | 하단 오버레이 채팅 패널 (슬라이드 업/다운 애니메이션) |
| `ChatInputBar.tsx` | 최소화 상태의 채팅 입력 바 (하단 고정) |

### 8.2 기존 컴포넌트 재활용

| 컴포넌트 | 변경 |
|----------|------|
| `MessageList` | 변경 없음 — ChatOverlay 내부에서 사용 |
| `ChatInput` | 변경 없음 — ChatOverlay, ChatInputBar 양쪽에서 사용 |
| `MessageBubble` | 변경 없음 |
| `WidgetRegistry` | 변경 없음 — 위젯 렌더링 로직 그대로 (배치 위치만 달라짐) |
| `AIProvider` | AIMode 타입에 `"native"` 추가. 캔버스 페이지 상태 관리 추가 |
| `AppLayout` | native 모드 분기 렌더링 추가 |
| `AIStatusChip` | 모드 순환에 native 추가 |

### 8.3 렌더링 분기 (AppLayout)

```
AppLayoutInner
  ├── IF native 모드 + isOpen:
  │   └── AINativeMode
  │       ├── CanvasArea
  │       │   ├── CanvasPage (현재 활성 페이지)
  │       │   │   ├── CanvasWidget × N
  │       │   │   └── EmptyPlaceholder (위젯 없을 때)
  │       │   └── PageIndicator (● ○ ○ + 화살표)
  │       ├── ChatInputBar (하단 고정)
  │       └── ChatOverlay (포커스 시 슬라이드 업)
  │           ├── DimOverlay
  │           ├── ChatHeader (세션 드롭다운)
  │           ├── MessageList
  │           └── ChatInput
  ├── IF fullscreen 모드 → AIFullScreen (기존)
  ├── ELSE → Outlet (기존 페이지)
  ├── AISidePanel (side 모드, 기존)
  └── AIFloating (floating 모드, 기존)
```

## 9. 데이터 흐름

```
사용자 메시지 입력 (ChatOverlay)
  → AIProvider.sendMessage()
  → useAIChat SSE 스트리밍
  → AI 응답에 tool_result 포함 (canvas 필드 있음)
  → AIProvider가 모드 확인:
      ├── native:
      │   ├── canvas.page = "new" → 새 CanvasPage 생성 + 위젯 배치 + 자동 전환
      │   └── canvas.page = "current" → 현재 페이지에 위젯 추가/교체
      └── 기타 → 기존 인라인 렌더링
  → CanvasArea 리렌더 → CanvasPage → CanvasWidget 표시
  → PageIndicator 업데이트
```
