# AI 상태 칩 — UX 디자인 스펙

## 개요

AI 어시스턴트 진입점을 사이드바 버튼에서 **메인 콘텐츠 상단 중앙의 플로팅 상태 칩**으로 교체한다. 최소 공간으로 AI의 존재감을 유지하면서, 상태 표시/모드 전환/빠른 입력을 하나의 컴포넌트에 통합한다.

### 배경

- **기존 FAB (우하단 플로팅 버튼)**: 존재감은 좋았지만 콘텐츠/UI를 가림
- **기존 LNB (사이드바 버튼)**: 가리는 문제는 해결했지만 존재감, 발견성, 접근성이 떨어짐
- **전략적 방향**: AI를 "핵심 동반자 → AI-First"로 전환 예정. AI를 부가 기능이 아닌 플랫폼 중심 경험으로 설계해야 함

### 목표

- 항상 보이되 공간은 최소화 (칩 ~160x32px)
- AI 상태를 실시간으로 표시 (대기/응답 중/도구 실행/에러)
- 클릭 한 번으로 모드 전환 (로테이션)
- 호버로 상세 정보 확인 + 빠른 액션 수행
- AI-First 전환 시 자연스럽게 확장 가능한 구조

## 1. 상태 칩 컴포넌트

### 1.1 위치 및 레이아웃

- **위치**: 메인 콘텐츠 영역 상단 중앙, 상단 가장자리에서 8px 아래
- **z-index**: 메인 콘텐츠 위, 모달/드롭다운 아래
- **position**: `fixed` 또는 `absolute` (메인 콘텐츠 영역 기준)
- 사이드바와 겹치지 않음 (사이드바 영역 제외한 중앙)

### 1.2 칩 상태별 표시

#### 대기 — 세션 없음
```
┌─────────────────────────┐
│ ✦  AI 어시스턴트    🟢  │
└─────────────────────────┘
```
- 배경: `rgba(129,140,248,0.15)` 그라데이션
- 테두리: `rgba(129,140,248,0.3)` 1px
- border-radius: 20px
- 패딩: 6px 16px
- 녹색 점: `#4ade80`, 6x6px

#### 대기 — 세션 활성
```
┌──────────────────────────────┐
│ ✦  화재통계 분석  ·  3건  🟢 │
└──────────────────────────────┘
```
- 세션명 표시 (최대 10자, 초과 시 말줄임)
- 구분자 `·` (`rgba(129,140,248,0.4)`)
- 대화 건수

#### 응답 생성 중
```
┌────────────────────────────┐
│ ✦  응답 생성 중    ▰▰▱    │
└────────────────────────────┘
```
- ✦ 아이콘: 펄스 애니메이션 (opacity 1 ↔ 0.4, 1.5s)
- 프로그레스 바: 20px 폭, 슬라이딩 애니메이션
- 배경/테두리 밝기 증가
- 글로우 효과: `box-shadow: 0 0 12px rgba(129,140,248,0.2)`

#### 도구 실행 중
```
┌────────────────────────────┐
│ ⚡  데이터 조회 중   ▰▱▱   │
└────────────────────────────┘
```
- 아이콘: ⚡ (✦ 대신)
- 색상 체계: 노란색 (`#fbbf24` 계열)
- 배경: `rgba(251,191,36,0.15)`
- 테두리: `rgba(251,191,36,0.3)`
- 도구명 표시 (최대 8자)

#### 사이드 모드
```
┌──────────────────────────┐
│ ✦  화재통계 분석  ·  📊  │
└──────────────────────────┘
```
- 📊: 분할 사각형 아이콘 (SVG, 16x16 사각형 + 세로선)
- 배경/테두리 밝기 한 단계 증가
- 테두리: `#818cf8` solid

#### 풀스크린 모드
```
┌──────────────────────────┐
│ ✦  화재통계 분석  ·  ▢   │
└──────────────────────────┘
```
- ▢: 채움 사각형 아이콘 (SVG, 내부 fill 있음)
- 배경: `#818cf8 → #6366f1` 그라데이션 (fully opaque)
- 텍스트/아이콘: 흰색
- `box-shadow: 0 2px 8px rgba(129,140,248,0.3)`

#### 에러
```
┌─────────┐
│ ✦   !   │
└─────────┘
```
- 색상 체계: 빨간색 (`#f87171` 계열)
- 배경: `rgba(239,68,68,0.2)`
- 테두리: `rgba(239,68,68,0.4)`

### 1.3 칩 크기 가이드

| 상태 | 대략적 크기 |
|------|------------|
| 대기 (세션 없음) | ~160 x 32px |
| 대기 (세션 있음) | ~200 x 32px |
| 응답/도구 실행 | ~180 x 32px |
| 사이드/풀스크린 | ~190 x 32px |
| 에러 | ~70 x 32px |

## 2. 인터랙션

### 2.1 클릭 — 모드 로테이션

클릭할 때마다 모드가 순환:

```
닫힘 → 사이드 패널 → 풀스크린 → 닫힘
```

- 각 전환은 기존 `AIProvider`의 `setMode()` + `toggleAI()` 활용
- 전환 시 칩 상태가 즉시 변경 (아이콘, 색상, 배경)
- 애니메이션: 칩 자체는 `transition: all 200ms ease` 적용

### 2.2 호버 — 상태 카드 드롭다운

칩에 마우스를 올리면 아래로 상태 카드가 펼쳐짐.

#### 열기/닫기 규칙
- **열기**: 마우스 호버 시 200ms 딜레이 후 표시
- **유지**: 칩 또는 드롭다운 영역 내에 마우스가 있으면 유지
- **닫기**: 칩과 드롭다운 모두에서 마우스가 벗어나면 300ms 딜레이 후 닫힘
- **빠른 입력 중**: 입력창에 포커스가 있으면 마우스 이동과 관계없이 유지, ESC 또는 전송 시 닫힘
- **모바일**: 터치 시 열림, 바깥 터치 시 닫힘

#### 호버 카드 구성 — 닫힘 상태

```
┌─────────────────────────────────┐
│ ✦ AI 어시스턴트         🟢 대기 중 │
├─────────────────────────────────┤
│ 현재 세션                        │
│ 화재통계 분석       3건 · 2분 전   │
│ 토큰  ▰░░░░░░░░░  2.1k / 200k  │
├─────────────────────────────────┤
│ ┌─────────────────────────┐ ↵  │
│ │ 빠른 질문...              │    │
│ └─────────────────────────┘    │
├─────────────────────────────────┤
│  📊 사이드 열기   ▢ 풀스크린     │
│  ＋ 새 세션       ⏱ 세션 목록    │
├─────────────────────────────────┤
│        ⌘K 사이드 토글            │
└─────────────────────────────────┘
```

- **상태 영역**: AI 상태 표시 (대기/응답 중/도구 실행)
- **세션 정보**: 세션명, 대화 수, 마지막 활동, 토큰 사용량 프로그레스 바
- **빠른 입력**: 간단한 질문 즉시 전송 → 전송 시 사이드 패널 자동 오픈
- **액션 버튼**: 2x2 그리드 — 사이드 열기, 풀스크린, 새 세션, 세션 목록
- **키보드 힌트**: ⌘K 단축키 안내

#### 호버 카드 구성 — 사이드/풀스크린 상태

```
┌─────────────────────────────────┐
│ ✦ AI 어시스턴트         🟢 대기 중 │
├─────────────────────────────────┤
│ 현재 세션                        │
│ 화재통계 분석       3건 · 2분 전   │
│ 토큰  ▰░░░░░░░░░  2.1k / 200k  │
│ ┌─────────────────────────────┐ │
│ │ AI: 서울시 화재통계 데이터셋은... │ │
│ └─────────────────────────────┘ │
├─────────────────────────────────┤
│ 모드 전환                        │
│  ✕ 닫기   [📊 사이드]   ▢ 풀스크린 │
│                                 │
│  ＋ 새 세션       ⏱ 세션 목록    │
├─────────────────────────────────┤
│   ⌘K 사이드 토글 │ 클릭 모드 순환  │
└─────────────────────────────────┘
```

- **빠른 입력 제거**: 이미 패널이 열려있으므로 불필요
- **마지막 응답 미리보기**: AI 마지막 응답 첫 줄 표시
- **모드 전환**: 3칸 그리드, 현재 모드 하이라이트 (border 강조)
- **닫기 버튼**: 모드 전환에 "닫기" 옵션 포함

#### 호버 카드 스타일

- 크기: 320px 폭
- 배경: `#1e1e36`
- 테두리: `rgba(129,140,248,0.3)` 1px
- border-radius: 14px
- 그림자: `0 12px 40px rgba(0,0,0,0.5)`
- backdrop-filter: `blur(12px)`

### 2.3 키보드 단축키

- **⌘K (Cmd+K)**: 사이드 패널 토글 (닫힘 ↔ 사이드)
- 기존 동작과 동일, 변경 없음

## 4. 컴포넌트 구조

### 4.1 새로 생성

```
src/components/ai/
├── AIStatusChip.tsx          # 상태 칩 본체
├── AIStatusChipDropdown.tsx  # 호버 시 드롭다운 패널
└── (기존 파일 유지)
```

### 4.2 AIStatusChip.tsx

**Props**: 없음 (AIProvider context 사용)

**Context에서 사용하는 값**:
- `isOpen`: AI 패널 열림 여부
- `mode`: 현재 모드 (`'side' | 'fullscreen'`)
- `isStreaming`: 응답 생성 중
- `isThinking`: 도구 실행 중
- `currentSessionId`: 현재 세션
- `messages`: 대화 메시지 목록

**내부 상태**:
- `isHovered`: 호버 여부
- `showDropdown`: 드롭다운 표시 여부 (딜레이 적용)

**렌더링 로직**:
1. 현재 상태 판단 (대기/응답/도구/에러/사이드/풀스크린)
2. 상태에 따른 아이콘, 텍스트, 색상 결정
3. 칩 렌더링
4. 호버 시 `AIStatusChipDropdown` 렌더링

**클릭 핸들러**: 모드 로테이션
```
if (!isOpen) → setMode('side'), open
else if (mode === 'side') → setMode('fullscreen')
else if (mode === 'fullscreen') → close
```

### 4.3 AIStatusChipDropdown.tsx

**Props**:
- `isOpen`: AI 패널 열림 여부
- `mode`: 현재 모드
- `onModeChange`: 모드 변경 콜백
- `onNewSession`: 새 세션 생성 콜백
- `onClose`: 드롭다운 닫기 콜백

**섹션별 렌더링**:
1. 헤더 (AI 상태)
2. 세션 정보 (세션명, 대화 수, 토큰)
3. 빠른 입력 (닫힘 상태만) 또는 마지막 응답 미리보기 (열림 상태)
4. 액션 버튼 그리드
5. 키보드 힌트 푸터

## 5. 상태 관리

### 5.1 AIProvider 변경

```typescript
// Before
type AIMode = 'side' | 'floating' | 'fullscreen';

// After
type AIMode = 'side' | 'fullscreen';
```

기존 `toggleAI()`, `setMode()` 인터페이스는 유지. 칩 컴포넌트가 이를 조합하여 로테이션 구현.

### 5.2 localStorage

- `ai-mode`: 유지 (기본값 `'side'`)
- `ai-floating-pos`: 제거

## 6. 애니메이션 및 트랜지션

| 대상 | 효과 | 속도 |
|------|------|------|
| 칩 상태 전환 | 배경색, 테두리, 그림자 트랜지션 | 200ms ease |
| 드롭다운 열기 | opacity 0→1 + translateY(-4px→0) | 150ms ease-out |
| 드롭다운 닫기 | opacity 1→0 | 100ms ease-in |
| ✦ 펄스 (응답 중) | opacity 1 ↔ 0.4 | 1.5s ease-in-out infinite |
| 프로그레스 바 | translateX 슬라이딩 | 1s ease-in-out infinite |

## 7. 반응형 / 접근성

### 7.1 반응형

- **데스크톱**: 전체 기능 (호버 드롭다운 포함)
- **모바일**: 호버 → 탭으로 대체, 드롭다운은 탭으로 열고 바깥 탭으로 닫기
- **사이드바 접힘 시**: 칩 위치는 메인 콘텐츠 영역 기준 중앙 유지

### 7.2 접근성

- 칩: `role="button"`, `aria-label="AI 어시스턴트 - 현재 상태: {상태}"`
- 드롭다운: `role="menu"`, 포커스 트랩 적용
- 키보드: Tab으로 칩 포커스 → Enter로 클릭 → Escape로 드롭다운 닫기
- 스크린리더: 상태 변경 시 `aria-live="polite"`로 알림
