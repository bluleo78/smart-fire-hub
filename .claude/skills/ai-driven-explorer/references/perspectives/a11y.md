# Accessibility 관점 — WCAG 2.2 AA 검증

매트릭스 파일: `.coverage-matrix-a11y.md`.

> 출처: WCAG 2.2 · WAI-ARIA Authoring Practices Guide · Radix Primitives Accessibility · axe-core 4.10. 권위 출처는 §5.

이 perspective는 **장애 사용자가 실제로 사용 가능한가**를 본격 검증한다. axe-core 같은 자동 도구로 잡히는 항목(색대비 수치, ARIA 속성 유효성)은 의도적으로 제외하고, **사람(또는 AI agent의 의미적 판단)이 봐야만 잡히는 것**에 집중.

## 0. 자동 vs 수동 분리 원칙

| 도구 | 적합 영역 | 본 체크리스트 |
|---|---|---|
| axe-core / pa11y | 색대비, ARIA 속성 유효성, lang 존재, alt 존재, 중복 ID | 제외 (자동 위임) |
| Lighthouse a11y | 위 + 폼 레이블 누락, 버튼 이름 비어있음 | 제외 |
| **AI + playwright-cli (본 체크리스트)** | 키보드 흐름, 포커스 시각화, 동적 변경 안내, 의미적 라벨, 한국어 발음, prefers-reduced-motion, WCAG 2.2 신설 6항목 | **포함** |
| Sense Reader / NVDA 수동 | 실제 음성 청취 | 부분 (DOM 검증만, 음성은 사람) |

원칙: **"axe로 충분한 것은 안 본다. axe가 못 잡는 의미·흐름·맥락만 본다."**

## 2. 카테고리별 시나리오 (수동 24개)

### 2.1 키보드 조작 (4)
- [ ] **K1. Tab 도달**: 모든 인터랙티브 요소가 Tab만으로 도달. `<div onClick>`이 없는지. (SC 2.1.1)
- [ ] **K2. 키보드 트랩 부재**: 모달/임베드 안에서 Tab/Shift+Tab/Esc로 빠져나올 수 있다. (SC 2.1.2)
- [ ] **K3. 단일문자 단축키**: `/` 검색 같은 단축키가 입력 필드 포커스 중에도 트리거되지 않거나 끄거나 변경 가능. (SC 2.1.4)
- [ ] **K4. APG 패턴 키**: Dialog는 Esc, Menu/Listbox는 ↑/↓ + Home/End, Tabs는 ←/→, Combobox는 Alt+↓. shadcn/Radix는 통과해야 정상.

### 2.2 포커스 관리 (4)
- [ ] **F1. 시각 포커스 링**: 명확한 시각 표시(2px+ outline/ring). Tailwind `focus-visible:` 또는 대체 스타일 존재. 배경 대비 3:1 이상. (SC 2.4.7)
- [ ] **F2. 포커스 가림 ⭐**: sticky 헤더/플로팅 푸터/Toast가 키보드 포커스를 **완전히** 가리지 않음. (SC 2.4.11)
- [ ] **F3. 모달 포커스 복귀**: 모달 열림 → 내부로 포커스, 닫힘 → 트리거로 복귀. 트리거 사라진 경우 합리적 대체.
- [ ] **F4. 동적 영역 포커스 점프 금지**: Toast/알림/일부 새로고침 시 자동 포커스 강탈 X. (SC 3.2.1)

### 2.3 스크린리더 의미 (3)
- [ ] **SR1. 아이콘 버튼 이름**: shadcn `<Button size="icon">`에 `aria-label` 또는 `<span className="sr-only">` 존재. playwright `getByRole("button", { name })` 매칭. (SC 4.1.2 + 2.5.3)
- [ ] **SR2. 동적 변경 announce**: 검색 결과 개수, 폼 검증 에러, 비동기 로딩 완료가 `role="status"`/`role="alert"`/`aria-live`로 안내. (SC 4.1.3, 3.3.1)
- [ ] **SR3. 헤딩 구조**: `<h1>` 페이지당 1개, 섹션 시작 `<h2>`, h3 건너뛰지 않음. Radix `DialogTitle`이 `<h2>`로 자동 렌더되는지.

### 2.4 한국어 / 다국어 (3)
- [ ] **L1. `lang="ko"` 루트** 설정. (SC 3.1.1, 자동이지만 누락 빈번)
- [ ] **L2. 부분 lang ⭐ 한국어 함정**: 영문 약어/제품명 혼용("AI 분석", "Smart Fire Hub", "PostgreSQL")이 `<span lang="en">` 없이 길게 나오면 한국어 SR이 자모 단위로 잘못 발음. (SC 3.1.2)
- [ ] **L3. IME 조합 중 폼 검증**: 한글 입력 조합 중 `onChange`로 즉시 검증·자동 제출이 발생하지 않는지. `compositionstart/end` 처리 누락 시 "ㅇ안녕"에서 검증 트리거. (SC 3.2.2)

### 2.5 비주얼 / 모션 (3)
- [ ] **V1. 색만으로 정보 전달 금지**: 에러 빨강/성공 초록만이 아닌 아이콘·텍스트 병행. (SC 1.4.1)
- [ ] **V2. Reflow & 텍스트 간격**: WCAG 1.4.10은 320 CSS px 폭에서 가로 스크롤 없음을 요구하지만, 본 프로젝트(데스크탑 전용)에서는 1280px 좁은 노트북 폭에서 가로 스크롤·콘텐츠 잘림 없음을 우선 검증. line-height 1.5/letter-spacing 0.12em 강제 시 텍스트 잘림 없음은 폭과 무관하게 검증. (SC 1.4.10, 1.4.12 — 데스크탑 SaaS는 320px 의무 완화 가능, 단 키보드/SR 접근만 보장되면 OK)
- [ ] **V3. prefers-reduced-motion**: OS/Chromium emulation 활성 시 비기능 모션이 즉시 표시 또는 짧은 페이드로 대체. Tailwind v4 `motion-reduce:` 또는 미디어쿼리.

### 2.6 폼 / 입력 (4)
- [ ] **FO1. Label in Name**: visible 텍스트가 "비밀번호"인데 `aria-label`이 "Password"면 음성 명령 사용자 매칭 실패. visible label이 accessible name에 포함되어야 함. (SC 2.5.3)
- [ ] **FO2. 에러 식별 + 제안 ⭐**: (a) 어떤 필드 (b) 무엇이 틀렸는지 (c) 수정 예시 — 3요소 텍스트로 + `aria-describedby`로 입력 연결. (SC 3.3.1, 3.3.3)
- [ ] **FO3. Redundant Entry ⭐**: 다단계 폼에서 이전 단계 입력(이메일/이름/주소)을 재입력 요구하지 않음. (SC 3.3.7)
- [ ] **FO4. 인증 ⭐**: CAPTCHA 단독 의존, 인지 테스트, 마우스 정확도 퍼즐이 없거나 대안 제공. SSO/매직링크/패스키는 통과. (SC 3.3.8)

### 2.7 터치 / 포인터 (3)
- [ ] **T1. Target Size 24×24 ⭐**: 모든 클릭 가능 요소 hit area 최소 24×24 CSS px. 인라인 텍스트 링크 예외. (SC 2.5.8)
- [ ] **T2. Dragging 대안 ⭐**: 칸반·슬라이더·정렬·지도 panning에 single-pointer 대안(클릭 메뉴, 화살표 버튼, 키보드). (SC 2.5.7)
- [ ] **T3. Pointer cancellation**: 버튼이 mousedown 즉시 동작하지 않고 click(mouseup)에서 동작 — 잘못 누른 후 드래그-아웃으로 취소 가능. (SC 2.5.2)

## 3. Radix 의존 컴포넌트 검증 — 자동/수동 경계

shadcn/ui는 Radix 기반. **Radix가 자동으로 처리하는 것**과 **개발자 책임**의 경계가 핵심 함정.

### 3.1 Radix가 보장 (재검증 불필요)
- Dialog/AlertDialog/Popover의 focus trap과 Escape 닫기
- 열림 시 첫 포커스, 닫힘 시 트리거 복귀
- `aria-modal`, `role="dialog"`, `aria-haspopup`, `aria-expanded` 자동 부여
- Tabs/Menu/Listbox 화살표 키 이동, roving tabindex
- 모달 외부 inert 처리

### 3.2 Radix가 보장하지 않음 — 개발자 책임 (반드시 체크)

| 항목 | 흔한 실수 | 검증 |
|---|---|---|
| **DialogTitle/Description 누락** | `<DialogTitle>` 없이 `<DialogContent>` 사용. 콘솔 경고는 뜨지만 빌드는 통과 | DOM `[role=dialog]`의 `aria-labelledby`가 실제 텍스트 노드를 가리키는지. 시각적 숨김은 `<VisuallyHidden>` |
| **아이콘 트리거 라벨** | `<Button size="icon"><MoreIcon/></Button>` — Radix는 ARIA 깔지만 **이름은 안 만들어줌** | `aria-label="옵션 더보기"` 또는 `<span className="sr-only">` |
| **asChild 중첩 인터랙티브** | `<TooltipTrigger asChild><Button>` 안에 또 `<Link>` — 중첩 버튼/링크 충돌 | Tab 키로 두 번 멈추는지 |
| **Toast announce 정책** | Radix Toast `role="status"` 기본이지만 긴급 에러는 `role="alert"`/`type="foreground"` 명시 필요 | Toast의 `role` 속성 |
| **Tooltip dismissible** | Tooltip이 hover로만 떠 있고 키보드 사용자가 Esc로 못 닫는 경우 — Radix Tooltip은 OK지만 커스텀 hover 패널은 망가짐 (SC 1.4.13) | Tab으로 트리거 포커스 → Esc 눌렀을 때 닫히는지 |
| **Combobox / Command 라벨** | shadcn `Command` 검색 입력에 `aria-label` 또는 시각 라벨 누락 | `[role=combobox]`의 accessible name |
| **Nested 모달 inert** | 모달 안에서 Sheet 또는 두 번째 Dialog 열 때 첫 모달 trap이 살아있는지 | 두 단계 모달에서 Tab 순환이 가장 위 모달에 한정되는지 |
| **Form 에러 연결** | `<FormMessage>`가 `aria-describedby`로 자동 연결되지만, 커스텀 에러 텍스트를 다른 위치에 두면 끊어짐 | 입력의 `aria-describedby`가 실제 에러 노드 ID와 매칭 |
| **`onCloseAutoFocus` 오버라이드** | `e.preventDefault()`로 기본 포커스 동작 막은 후 직접 포커스 안 주면 `<body>`로 떨어짐 | 모달 닫은 직후 `document.activeElement` |
| **`outline-none` 남용** | shadcn 기본의 `focus-visible:ring`을 지우거나 추가 변형에서 누락 | 모든 인터랙티브 컴포넌트의 포커스 시 ring/outline 시각 |

### 3.3 shadcn 특화
- shadcn은 컴포넌트를 **코드로 복사**해 쓰므로 프로젝트마다 a11y 회귀 가능. Radix 업스트림이 고친 버그가 복사된 코드엔 그대로 남을 수 있다.
- `cn()` 헬퍼로 클래스 병합 시 `focus-visible:ring`이 후행 className에 의해 제거되는지.
- `Sheet`(슬라이드 패널)는 내부적으로 Dialog지만, 모바일에서 swipe로 닫는 경우 키보드 사용자에게 동등 동작(Escape) 보장.

## 4. AI 탐색 패스 통합 가이드

playwright-cli로 a11y 패스를 돌릴 때 다음 순서:

1. **준비**: `<html lang>`, axe 자동 스캔 1회 → 자동 항목 거름
2. **키보드 패스**: Tab으로 끝까지 이동, 각 stop 스크린샷 + accessibleName → K1, F1, F2, SR1
3. **모달/메뉴 패스**: 모든 trigger enumerate → 하나씩 열고 Esc 닫기, 포커스 복귀 → K2, K4, F3, §3.2 항목
4. **폼 패스**: 빈 제출 → 에러 ARIA 검증, 부분 입력 → 검증 타이밍 → FO1~FO4
5. **언어 패스**: 영문 비율 ≥30% 영역의 부모 `lang` 속성 → L2
6. **모션 패스**: Chromium emulation `prefers-reduced-motion: reduce` 토글 → 차이 캡처 → V3
7. **반응형 패스**: 320px → 가로 스크롤, sticky 가림 → V2, F2
8. **터치 패스**: 모든 button/a의 BoundingBox.width/height → 24×24 미달 → T1

각 결함 발견 시 보고:
```
[a11y/CATEGORY] WCAG {SC#} {제목}
- 위치: {URL} {selector}
- 현상: {스크린샷 / DOM 스니펫}
- 영향: {SR 사용자 / 키보드 사용자 / 저시력 / 한국어 SR}
- 권장 수정: {Radix prop / aria-label / motion-reduce 변형}
```

## 6. 의도적으로 제외

### 6.1 axe-core / pa11y 자동 검출 위임
- 색대비 4.5:1 / 3:1 (SC 1.4.3, 1.4.11)
- `<html lang>` 존재 (SC 3.1.1)
- 이미지 `alt` 존재 (단, alt가 적절한지는 의미 검증 별도)
- 폼 input의 label 연결 (SC 1.3.1, 4.1.2)
- 중복 ID, ARIA 유효성, role 적합성

### 6.2 사람만 가능 (SR 청취 / 인지)
- Sense Reader/NVDA 실제 음성 자연스러움
- 인지장애 사용자 텍스트 명확성
- 색맹/저시력 실사용 체감
- 한국어 SR의 영문 약어 발음 청취 (DOM은 lang 속성만 확인 가능)

### 6.3 너무 깊은 사례
- ARIA 1.3 신규 role
- Tree/Treegrid/Grid의 2D 키 네비
- Web Components Shadow DOM 내부 ARIA
- WCAG AAA 항목
